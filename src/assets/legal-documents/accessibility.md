## 1. Commitment to Accessibility

ReadingDaily Scripture App is committed to ensuring digital accessibility for people with disabilities. We continually improve our app to meet or exceed accessibility standards including:

- **WCAG 2.1 Level AA** - Web Content Accessibility Guidelines
- **Section 508 of the Rehabilitation Act**
- **Americans with Disabilities Act (ADA)**
- **EU Accessibility Directive 2016/2102**

---

## 2. Accessibility Features

### 2.1 Visual Accessibility

**Font and Text:**
- Adjustable font sizes (small to large)
- High contrast text options
- Support for system font scaling
- Clear, legible typography

**Color and Contrast:**
- Minimum 4.5:1 contrast ratio for normal text
- Minimum 3:1 contrast ratio for large text
- Dark mode and light mode support
- Non-color-dependent indicators

**Layout and Navigation:**
- Large touch targets (minimum 44x44 points)
- Consistent navigation patterns
- Adequate spacing between interactive elements
- Clear visual hierarchy

### 2.2 Screen Reader Support

The app is compatible with:
- **iOS:** VoiceOver
- **Android:** TalkBack
- **Full accessibility labels** for all interactive elements
- **Semantic HTML structure** for web components
- **Image descriptions** for visual content
- **Link text clearly describes destination**

### 2.3 Audio Accessibility

**Captions and Transcripts:**
- Optional captions for audio content
- Downloadable transcripts of scripture readings
- Timestamps for audio segments

**Audio Controls:**
- Adjustable playback speed (0.5x to 2.0x)
- Pause and resume functionality
- Audio descriptions of key actions
- Configurable audio levels

### 2.4 Motor Accessibility

**Touch and Gesture:**
- Large buttons and touch targets
- Alternative tap options for complex gestures
- Customizable gesture sensitivity
- Support for assistive touch devices

**Keyboard Navigation:**
- Full keyboard navigation support (where applicable on mobile)
- Logical tab order
- Keyboard shortcuts for common actions
- Focus indicators clearly visible

### 2.5 Cognitive Accessibility

**Clarity and Simplicity:**
- Clear, plain language (avoiding jargon)
- Consistent terminology
- Logical information organization
- Predictable navigation patterns

**User Control:**
- Ability to pause and adjust animation speeds
- Disable auto-playing content
- Confirmation before destructive actions
- Clear error messages with suggestions

### 2.6 Language and Localization

- Support for multiple languages
- Clear language selection
- Translation accessibility options
- Right-to-left language support

## 3. Known Accessibility Limitations

While we strive for full accessibility, some limitations may exist:

### 3.1 Audio Content

- Scripture audio is not currently provided with professional captions
- Some background elements in audio may be unclear to screen readers
- Real-time audio transcription has accuracy limitations

**Workaround:** Download text transcripts of readings instead

### 3.2 Complex Gestures

- Some advanced gestures in offline mode may not have keyboard alternatives
- Swipe navigation is primary, tap alternatives available

**Workaround:** Use tap-based navigation in settings

### 3.3 Third-Party Services

- Google Cloud Speech-to-Text has its own accessibility features
- Firebase services are subject to Google's accessibility commitments
- Third-party payment processors may have limitations
